## 1. Mission

Transform this repository from a 61-page retail-dashboard demo into **an AI-native hedge fund platform**: a small number of deep, fully-functional surfaces built around one core loop — **AI agents autonomously research, debate, risk-manage, and paper-trade a real portfolio on a schedule, with every decision fully auditable.**

The product is **PortAI**. The fund engine is the **Stratton Fund**. We are NOT starting fresh — the existing `backend/app/stratton/` engine is good and gets reused. Everything around it gets rebuilt or deleted.

**North-star qualities:** real data everywhere (zero mock/placeholder anywhere in the shipped app), fast (aggressive caching, one LLM call per run), auditable (every trade traceable to agent signals), and honest (README claims only what works, backed by measured numbers).

## 3. Product Vision — the 6 Surfaces

The entire frontend becomes exactly six pages. Every one fully real.

| # | Page | Route | What it does |
|---|------|-------|--------------|
| 1 | **Fund Console** | `/` | The fund at a glance: NAV curve vs benchmark (NIFTY 50 via `^NSEI`, toggle SPY), current positions with live P&L and mini-charts, cash, today's orders, next scheduled run countdown, kill-switch toggle. |
| 2 | **Decision Room** | `/decisions` and `/decisions/[runId]` | The audit trail. List of every fund run. Detail view = the "agent debate": per ticker, every agent's signal (bull/bear/neutral + confidence + factors), the risk manager's verdict, the PM's final orders, and the AI-written memo. This page is the soul of the product. |
| 3 | **Research Terminal** | `/research` and `/research/[ticker]` | On-demand deep-dive on any ticker: full candlestick chart (existing AdvancedChart with indicators), agent panel (run the pipeline for just this ticker, stream results), fundamentals snapshot, recent news with sentiment. |
| 4 | **Backtest Lab** | `/backtest` | Run the same agent pipeline over a historical window with point-in-time data. Equity curve vs benchmark, metrics table (CAGR, Sharpe, max DD, win rate), trade log. |
| 5 | **Risk Desk** | `/risk` | Live portfolio risk: VaR (95%, historical), volatility, beta vs benchmark, max drawdown chart, per-position exposure vs caps, correlation heatmap of holdings, monthly returns heatmap. |
| 6 | **Settings** | `/settings` | Fund config (base currency, universe/watchlist, position cap %, run schedule, active personas as toggles), Telegram connection + test message, LLM provider selection, auth/profile. |

**The core loop (what makes it "AI-native"):** on a schedule (default: daily at market close), the fund runs automatically → prefetch data → all agents score the universe in parallel → risk manager aggregates → portfolio manager emits orders → orders execute against the paper ledger → a single LLM call writes a "fund memo" explaining the day's decisions → memo + orders saved to the decision store → memo pushed to Telegram. A human can trigger a run manually, pause the fund (kill-switch), or override the universe — but the fund runs itself.

### 4.2 Data layer spec (this is the single biggest efficiency win)

- **Provider protocol** (`data/providers/base.py`): async methods `get_quote(symbol)`, `get_ohlcv(symbol, interval, start, end)`, `get_financials(symbol, quarters=8)`, `get_news(symbol, limit=20)`, `get_fx(pair)`, `get_details(symbol)`. Normalized pydantic return models. `YFinanceAdapter` implements it (run yfinance's sync calls via `asyncio.to_thread`).
- **Read-through cache** (`data/cache.py`): every read checks a SQLite `market_cache` table first (key = provider+method+params hash, value = JSON, `fetched_at`, `ttl_seconds`). Miss → provider → store → return. **TTLs:** quote 60s · intraday OHLCV 5min · daily OHLCV for past dates = immutable (never refetch) · financials 24h · news 15min · fx 60s · details 7d.
- **Singleflight:** in-process `asyncio.Lock` per cache key so 15 agents / concurrent requests for the same ticker cause exactly ONE provider call.
- **Instrumentation:** counters for cache_hits, cache_misses, provider_calls, per-endpoint latency — exposed at `/api/v1/meta/stats`.
- **Symbols:** NSE tickers require the `.NS` suffix for yfinance (BSE = `.BO`). Store a small symbol resolver (user types `RELIANCE` → `RELIANCE.NS`; US tickers pass through). Benchmark symbols: `^NSEI`, `SPY`. FX: `USDINR=X`.

### 4.3 PersonaEngine spec

Replace the 12 copy-paste persona files with:
- `rules.py`: composable primitives, e.g. `metric_threshold(field, op, value, bull_pts, bear_pts, label)`, `trend(field, quarters, direction, pts, label)`, `price_vs_sma(window, …)`, `sentiment_score(…)`.
- `configs/*.yaml`: one per investor. Example (`buffett.yaml`): display name, philosophy blurb, and a list of rules — `return_on_equity > 0.15 → +2 bull "Durable ROE"`, `debt_to_equity > 2 → +2 bear "Excessive leverage"`, etc. **Port every rule from every existing persona file faithfully.**
- `engine.py`: loads configs, evaluates rules against `AgentState`, emits the same signal shape as today (`{agent, ticker, direction, confidence, factors[]}`).
- **Golden parity tests (mandatory):** before deleting the old persona files, record their outputs on fixture data (build fixtures from the existing tests in `stratton/tests/`). The PersonaEngine must reproduce identical signals for all 12 personas on those fixtures. Only then delete the old files.
- The 6 analyst agents (technical, sentiment, fundamentals, valuation, macro_regime, growth) stay as code — they compute real indicators and don't fit the config model. Clean them up, don't genericize them.

### 4.4 Pipeline (replaces LangGraph)

`engine/pipeline.py`:
```
async def run_pipeline(universe, personas, state_seed) -> RunResult:
    state = await prefetch(universe)                      # through cache
    signals = await asyncio.gather(*analysts, *persona_engine.evaluate_all())
    risk = risk_manager(state, signals)
    orders = portfolio_manager(state, risk)
    memo = await synthesis.write_memo(state, signals, risk, orders)   # ONE llm call, streamed if caller wants SSE
    return RunResult(signals, risk, orders, memo, timings, llm_cost)
```
- Remove `langgraph` from requirements. Write `docs/adr/001-remove-langgraph.md` explaining: static fan-out/fan-in topology, asyncio is simpler/faster/lighter, measured cold-start delta.
- `synthesis.py`: default provider Groq `llama-3.3-70b-versatile` (fall back through the factory if `GROQ_API_KEY` absent; if NO llm key exists, skip the memo gracefully and mark it `memo: null` — the deterministic pipeline must still fully work).
- **Analysis result cache:** full research-terminal analysis cached per `(sorted_tickers_hash, date)` in SQLite — a repeat lookup the same day returns instantly.

### 4.5 Fund loop spec

- **Fund model:** one fund per user (`user_id` from Supabase JWT; dev fallback user `"local"`). Fields: base_currency (default INR), starting_cash (default ₹10,00,000), universe (default: 8–10 liquid NSE names + 3–4 US megacaps), position_cap_pct (default 30), active_personas, schedule_cron (default weekdays 15:45 IST), is_paused.
- **Ledger:** evolve `paper_trader.py` → `fund/ledger.py` backed by SQLite tables `positions`, `orders`, `nav_snapshots`. Orders fill at latest cached quote. USD positions convert to base currency at cached USDINR for NAV.
- **Scheduler:** APScheduler (AsyncIOScheduler started in app lifespan). Job = `fund/service.py::run_fund(fund_id)`: pipeline → apply orders → snapshot NAV → persist run + all signals to `runs`/`signals` tables → Telegram memo (if configured). Manual trigger endpoint `POST /api/v1/fund/run`. Kill-switch = `is_paused` checked before every run.
- **Decision store:** `runs` (id, fund_id, ts, universe, latency_ms, llm_cost, memo) and `signals` (run_id, agent, ticker, direction, confidence, factors JSON) + `orders` rows linked to run_id. This powers the Decision Room with zero extra work.

### 4.6 Backtest spec

Reuse `stratton/src/backtest` metrics/tracker. Backtest = run the pipeline weekly over a historical window where prefetch is **point-in-time**: OHLCV sliced to `<= as_of`, financials filtered to quarters reported before `as_of`, news skipped (sentiment agent returns neutral in backtests). **State this in the UI: "Backtests are point-in-time on prices & fundamentals; news sentiment excluded to avoid lookahead bias."** Run as background task, poll or SSE progress, persist results.

## 6. Global Rules (apply in every phase)

1. **No mock data, ever.** If real data for something can't be fetched free, the feature doesn't ship. Degrade gracefully (e.g., no LLM key → no memo, pipeline still runs).
2. **No new feature pages.** Six pages. If a feature idea appears, it goes into `docs/LATER.md`, not the code.
3. **Don't break the engine's brain.** Persona rule logic and analyst math must be ported faithfully — parity tests are the contract. Improving thresholds is out of scope.
4. **Typed everywhere.** Pydantic models on every API boundary; TS strict; no `any` except at true library edges.
5. **Tests are part of the phase**, not an afterthought. A phase without its listed tests is incomplete.
6. **Conventional commits, one commit per coherent unit,** phase branch optional but commits must leave the repo working.
7. **Secrets:** only via env (`app/config.py`); never commit keys; keep `.env.example` updated each phase.
8. **Free-tier respect:** yfinance is unofficial — throttle provider calls (max ~5 concurrent, small jitter), rely on the cache, never tight-loop retries.

## 7. Known Pitfalls

- yfinance `.info` is slow/flaky, worse for `.NS` tickers — prefer `fast_info` for quotes and `.history()` for OHLCV; wrap sync calls in `asyncio.to_thread`; handle empty DataFrames (delisted/typo symbols) with a clean 404, not a 500.
- Indian market quirks: NSE trading hours 09:15–15:30 IST; schedule the fund run after close; `^NSEI` has no volume data — don't assume volume exists for indices.
- The existing stratton tests import via `from src...` — the relocation to `app.engine` will need import rewrites; do them mechanically and keep test intent identical.
- SQLite + async: use one engine, `aiosqlite`, and don't share sessions across tasks; WAL mode on.
- APScheduler + uvicorn reload runs jobs twice in dev — guard scheduler start behind a flag or disable reload for the scheduler process.
- SSE through Next.js dev proxy can buffer — hit the FastAPI origin directly from the browser client for streams (`NEXT_PUBLIC_API_URL`).
- Don't let react-query refetch intervals undercut cache TTLs (pointless load).
